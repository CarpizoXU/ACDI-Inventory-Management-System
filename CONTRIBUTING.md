# Contributing to ACDI Inventory Management System

## Contributor Workflow

1. Fork the repository.
2. Create a branch: `feature/<feature-name>` or `fix/<bug-name>`.
3. Write tests before implementation.
4. Keep changes focused and use clear commit messages.
5. Submit a pull request against `develop`.

## Coding Standards

- Use descriptive variable and function names.
- Keep modules small and single-purpose.
- Follow ESLint and Prettier formatting.
- Validate inputs and handle errors centrally.

## Testing

- Add unit tests for new features.
- Add integration tests for API endpoints.
- Keep coverage above 80% for core modules.

## Git Commit Format

- `feat:` for new features
- `fix:` for bug fixes
- `refactor:` for code improvements
- `test:` for test-related changes
- `docs:` for documentation updates
